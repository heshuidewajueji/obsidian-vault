# 📚 知识一体化 - 最佳实践指南

> 📅 创建日期: 2026-05-26
> 📌 用于: Obsidian 项目管理的知识整合指导

---

## 🎯 核心理念

> **任务笔记不是普通的 checkbox，是"微型项目文档"**

当一个任务完成后，它不应该只是被标记为"已完成"然后被遗忘——它应该成为你知识库中有价值的一笔财富。

---

## 📝 任务笔记的标准结构

```markdown
---
状态: "已完成"
项目: "你的项目名称"
任务: "任务标题"
截止日期: 2026-05-30
优先级: "高"
标签: []
负责人: King
预估: 4
实际: 3
创建日期: 2026-05-26
已完成: true
---

# 任务标题

## 📌 任务描述
[背景、目标、参考资料]

## ✅ 完成标准
- [x] 检查项 1
- [x] 检查项 2

## 📝 执行记录
### 进展更新

|------|------|------|

## 💡 经验总结
```

## 🔗 双向链接网络示意

```
                    ┌─────────────────┐
                    │   知识笔记      │
                    │ (学习笔记/文档) │
                    └────────┬────────┘
                             │ ↕ 双向链接
                             ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────────┐
│  项目笔记   │ ←→ │  任务笔记   │ ←→ │   知识笔记      │
│ (项目主页)  │     │ (今日任务)  │     │ (参考资料)      │
└─────────────┘     └─────────────┘     └─────────────────┘
       ↑                   │                   ↑
       │                   │                   │
       └───────────────────┴───────────────────┘
              ← 链接回 → 所属项目/分类
```

## 📋 链接层级示例

### 第一层：项目内部

```
TBOX项目主页.md
├── 链接 → 任务1.md
├── 链接 → 任务2.md
└── 链接 → 评审会议纪要.md
```

### 第二层：跨项目

```
TBOX项目主页.md
├── 链接 → [[汽车行业术语表]] （知识笔记）
└── 链接 → [[PPAP流程指南]] （知识笔记）
```

### 第三层：个人知识库

```
任务笔记.md
└── 链接 → [[30.areas/汽车行业/行业报告摘要]] （知识沉淀）
```

## ✅ 检查清单

在标记任务为"已完成"前，检查以下内容：

- [ ] 是否写明了任务背景和目标？
- [ ] 是否记录了关键的决策和原因？
- [ ] 是否添加了相关的知识笔记链接？
- [ ] 是否总结了这次任务的经验教训？
- [ ] 工作量记录是否准确？

## 📌 使用场景示例

### 场景1：TBOX 项目需求评审

```
任务笔记：TBOX需求评审准备.md
├── 📌 描述：准备评审材料，整理需求文档
├── ✅ 完成标准：[x] 整理需求文档 [x] 准备PPT [x] 预演
├── 📝 执行记录：记录了评审中发现的问题
├── 💡 经验总结：评审前应先与客户单独沟通
└── 🔗 链接：
    ├── [[TBOX项目主页]] （所属项目）
    ├── [[需求评审PPT模板]] （模板）
    └── [[评审话术指南]] （知识笔记）
```

### 场景2：学习新技术

```
任务笔记：学习 CAN 总线协议.md
├── 📌 描述：研究 CAN 总线协议原理
├── ✅ 完成标准：[x] 理解帧结构 [x] 理解优先级 [x] 实践示例
├── 💡 经验总结：CAN 总线适合短报文，高频场景
└── 🔗 链接：
    ├── [[CAN总线技术文档]] （外部文档）
    └── [[TBOX通信协议设计]] （应用场景）
```

## 🚀 下一步

当你有了真实项目后：

1. **创建项目笔记**：在 `30-📂项目管理/` 下创建项目主页
2. **创建任务笔记**：使用 `98-模板/模板-标准任务.md`
3. **建立链接**：每个任务都要链接回项目和相关知识
4. **定期回顾**：每月回顾完成的任务，更新知识笔记
