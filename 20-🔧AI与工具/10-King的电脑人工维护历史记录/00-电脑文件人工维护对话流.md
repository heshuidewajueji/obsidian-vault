## 问题4
`/Users/king-macbookair/.cc-switch` 这个文件夹里的文件也有类似用途，与`/Users/king-macbookair/.king-ai`可以整合吗？

## 解答4：

### 📊 两个目录功能对比

| 对比项 | `.king-ai/` | `.cc-switch/` |
|-------|-------------|---------------|
| **定位** | King AI 统一配置枢纽 | Claude Code Switch 多模型切换器 |
| **主配置文件** | `king.yaml` | `settings.json` |
| **数据库** | 多个 SQLite（记忆） | `cc-switch.db`（切换状态） |
| **Skills** | ✅ 70+ 个插件 | ✅ 71 个（符号链接模式） |
| **备份** | 无专门备份目录 | `backups/`（4个备份） |
| **日志** | 无 | `logs/cc-switch.log` |
| **认证** | `api-keys.md` | `copilot_auth.json` |

### 🔍 `.cc-switch` 详细结构

| 文件/目录 | 用途 |
|----------|------|
| `settings.json` | 主配置：语言、启动项、可见应用、当前提供商 |
| `cc-switch.db` | 数据库：~1.3MB，存储切换状态 |
| `copilot_auth.json` | Copilot 认证信息 |
| `skills/` | Skills 目录（与 `.king-ai/skills/` 同步） |
| `backups/` | 4个历史备份 |
| `skill-backups/` | Skills 备份 |

### ⚠️ 能否整合？**不建议**

**原因**：

1. **功能定位不同**
   - `.king-ai` → 用户配置（偏好、工作时间、禁忌）
   - `.cc-switch` → 运行时状态（当前模型、切换记录）

2. **数据性质不同**
   - `.king-ai` → 静态配置（很少变化）
   - `.cc-switch` → 动态状态（频繁读写）

3. **Skills 共用但独立管理**
   - 两个目录的 `skills/` 通过符号链接同步
   - 整合会导致循环引用或冲突

4. **备份机制独立**
   - `.cc-switch` 有自动备份（每天）
   - `.king-ai` 无备份机制

### ✅ 建议：保持分离，但可优化

| 优化项 | 说明 |
|-------|------|
| Skills 共享 | 已通过 symlink 共享，无需改动 |
| 备份整合 | 可在 `.king-ai/` 添加 `backups/` |
| 日志集中 | 可将 `.cc-switch/logs/` 移至统一位置 |
| 大小清理 | `.cc-switch/backups/` 可定期清理旧备份 |

### 📈 存储空间占用

| 目录 | 大小 | 主要占用 |
|------|------|---------|
| `.king-ai/` | ~75KB（不含 memory/*.sqlite） | Skills |
| `.cc-switch/` | ~4.8MB | cc-switch.db + 备份 |
| `.king-ai/memory/` | ~40MB | SQLite 数据库 |
