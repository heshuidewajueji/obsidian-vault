# 工程控制论 × Hermes Agent 监控框架

## 理论根基

钱学森《工程控制论》(1954) 的核心命题：

> 「如何从不可靠的组件构建可靠系统」

拒绝「必须消除所有缺陷」的传统思维，转向构建能够容忍和适应缺陷的系统架构。

## 八大控制论原理 → Agent 监控映射

| 控制论原理 | 工程含义 | 监控映射 |
|-----------|---------|---------|
| 反馈控制 | 用输出偏差驱动纠正 | 技能执行成功率追踪 |
| 系统稳定性 | 受扰后回归平衡态的能力 | 振荡检测 + MTTR |
| 黑箱方法 | 不看内部只看输入输出 | 技能 I/O 质量监控 |
| 信息论 | 区分信号与噪声 | 告警信噪比过滤 |
| 自适应控制 | 系统自调参数 | CUSUM 动态阈值 |
| 级联分析 | 故障如何传播 | 故障影响半径追踪 |
| 最优控制 | 资源最优分配 | 监控资源分配优化 |
| 系统辨识 | 从数据学习系统行为 | 技能行为基线建模 |

## 五层监控架构

```
L4 进化评估层  → 技能新鲜度 / 知识衰减率 / 架构漂移度
    频率: 每天 09:00
    指标: 过时技能数、平均年龄、新鲜度比率

L3 自适应控制层 → CUSUM 累积和漂移检测
    频率: 每 2 小时
    方法: 双边 CUSUM (k=0.5, h=5.0)
    原理: 检测微小但持续的均值偏移

L2 稳定性分析层 → 振荡检测 / 级联半径 / MTTR 趋势
    频率: 每 4 小时
    方法: 错误模式聚类 + 重复检测
    振荡阈值: 同类型错误 ≥ 5 次

L1 反馈信号层  → 技能成功率 / 错误类型分布 / 恢复率 / 延迟
    频率: 每 4 小时
    指标: 成功率、错误分布、恢复成功率、p50/p99

L0 组件存活层  → 进程心跳检测
    频率: 每 30 分钟
    方法: ps 进程检测
    组件: Hermes GW / OpenClaw GW / OpenClaw MCP / Hermes CLI
```

## 告警四级体系

| 级别 | 名称 | 控制论对应 | 响应策略 |
|------|------|-----------|---------|
| P0 | 紧急 | 系统崩溃 | 自动重启 + 即时通知 |
| P1 | 严重 | 正反馈失控 | 自动降级 + 5min 通知 |
| P2 | 警告 | 稳定裕度不足 | 自动记录 + 1h 通知 |
| P3 | 提示 | 慢变量漂移 | 仅记录 |

## 控制论设计原则

1. **反馈闭环强制**: 检测→告警→响应→验证 必须完整
2. **降级而非静默**: 监控自身部分失效时输出标注结果
3. **信噪比保护**: 单点异常不告警，连续/趋势异常才告警
4. **自适应阈值**: CUSUM 从历史学习基线，不硬编码
