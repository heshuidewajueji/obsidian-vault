## 一、指令系统概述

Claude Code 的核心交互通过**指令符号**完成，主要分为 6 类：

| 指令类型 | 符号 | 用途 | 使用频率 |
|----------|------|------|----------|
| **控制指令** | `/` | 控制 AI 行为、切换模式 | ⭐⭐⭐⭐⭐ |
| **引用指令** | `@` | 引用文件、目录、Git | ⭐⭐⭐⭐⭐ |
| **注释指令** | `#` | 添加说明、标记 | ⭐⭐⭐ |
| **查找指令** | `?` | 搜索历史、会话 | ⭐⭐⭐ |
| **权限指令** | `+` | 临时授予权限 | ⭐⭐ |
| **变量指令** | `$` | 设置临时变量 | ⭐⭐ |

## 二、控制指令 `/`

### 常用斜杠命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `/help` | 显示帮助信息 | `/help` |
| `/model` | 切换模型 | `/model opus` |
| `/plan` | 进入规划模式 | `/plan` |
| `/approve` | 批准待确认的操作 | `/approve` |
| `/exit` | 退出当前会话 | `/exit` |
| `/compact` | 压缩上下文 | `/compact` |
| `/clear` | 清屏 | `/clear` |

### 模式切换

| 命令 | 说明 |
|------|------|
| `/plan` | 进入规划模式，详细分析后执行 |
| `/continue` | 继续上次对话 |
| `/resum` | 恢复被中断的任务 |

## 三、引用指令 `@`

### 基本用法

`@` 用于引用各种资源，提供上下文给 Claude：

```bash
# 引用单个文件
@notes/my-note.md

# 引用目录
@src/

# 引用多个文件
@file1.md @file2.md @file3.md
```

### 引用类型

| 类型 | 语法 | 说明 |
|------|------|------|
| **文件** | `@path/to/file.md` | 引用文件内容 |
| **目录** | `@src/` | 引用目录下所有文件 |
| **Git** | `@git:branch` | 引用 Git 分支/提交 |
| **会话** | `@session:id` | 引用历史会话 |

### 实用示例

```
# 让 Claude 读取多个文件后总结
@project/config.json @project/README.md 请总结项目配置

# 让 Claude 理解整个目录结构
@src/ 分析这个模块的代码组织

# 引用特定行
@file.md:10-20 解释这些代码行
```

## 四、注释指令 `#`

### 用途

`#` 用于添加注释，不会被 Claude 执行：

| 用法 | 说明 |
|------|------|
| `# TODO` | 标记待办事项 |
| `# NOTE` | 添加备注 |
| `# SKIP` | 跳过某个文件 |
| `# PRIORITY:high` | 设置优先级 |

### 实用示例

```
帮我写一个函数 # 处理用户登录
# 这个函数需要处理验证码

# TODO: 完成后添加测试
# SKIP: node_modules/
```

## 五、查找指令 `?`

### 基本用法

| 命令 | 说明 |
|------|------|
| `? history` | 搜索对话历史 |
| `? files` | 搜索相关文件 |
| `? recent` | 查看最近会话 |

### 搜索历史

```
# 搜索包含关键词的历史
claude -r "数据库"

# 继续上次对话
claude -c
```

## 七、实用工作流

### 1. 新项目工作流

```
/plan                    # 启动规划模式
@README.md              # 提供项目说明
帮我分析项目结构         # 让 Claude 分析
/approve                # 批准计划
```

### 2. 快速修复工作流

```
@buggy-file.py           # 引用有问题的文件
修复这个 bug             # 直接让 Claude 修复
```

### 3. 代码审查工作流

```
/review                 # 使用代码审查技能
@src/                   # 审查整个目录
重点关注安全性问题
```

## 八、常见问题

| 问题 | 解决方案 |
|------|----------|
| Claude 不读取文件 | 使用 `@` 显式引用 |
| 上下文太长 | 使用 `/compact` 压缩 |
| 操作需要确认 | 使用 `/approve` |
| 想切换模型 | 使用 `/model sonnet` |

## 💡 今日要点

> 1. **`@` 是核心** - 需要什么就引用什么
> 2. **`/plan` 安全第一** - 复杂任务先规划
> 3. **`/help` 随时可用** - 忘了命令就问它
