# SQE基础课程：供应商质量管理入门

## 课程定位

**目标人群**：从TBOX项目经理转型SQE，前3个月快速建立基础能力

**学习目标**：3个月内掌握质量管理体系框架 + 供应商审核基础方法

**前置要求**：IATF 16949 / VDA 6.3 / ISO 26262基础知识（可通过AI教练框架快速建立认知）

---

## 学习路径

```
第1个月：质量管理体系框架（战略知识层）
第2个月：供应商审核方法（战术知识层）
第3个月：问题处理基础（实战验证层）
```

---

## 第1个月：质量管理体系框架

### 里程碑目标

- 能用自己的话解释IATF 16949的核心逻辑
- 能画出质量管理体系PDCA循环图
- 能识别零跑质量体系文件的关键模块

### 学习内容

#### W1-W2：IATF 16949核心条款速读

**学习任务**：
1. 用AI教练框架快速建立全局认知（参考：00-学习路径总览/SQE职业成长课程.md）
2. 精读核心条款：第4-10章（质量管理体系、设计开发、生产运营、服务）
3. 整理术语表：15-20个核心术语

**核心术语**：

| 术语 | 英文 | 理解要点 |
|------|------|----------|
| 质量方针 | Quality Policy | 质量方向性承诺，全员必须理解 |
| 质量目标 | Quality Objective | 可量化、可跟踪、与方针一致 |
| 管理评审 | Management Review | 定期评审体系有效性 |
| 内审 | Internal Audit | 自我发现问题的重要手段 |
| 不良品控制 | Nonconforming Control | 隔离、标识、评审、处置 |
| 纠正措施 | Corrective Action | 消除原因、防止再发 |
| 预防措施 | Preventive Action | 消除潜在原因、防止发生 |
| 过程方法 | Process Approach | 输入→活动→输出，强调接口 |
| 风险思维 | Risk-based Thinking | 机会与风险并重 |
| 持续改进 | Continual Improvement | PDCA永无止境 |

**验证方式**：能向非质量专业人员（如销售同事）解释3个核心条款的含义

#### W3：质量管理体系框架理解

**学习任务**：
1. 研究IATF 16949五大工具：APQP/PPAP/FMEA/MSA/SPC
2. 理解五大工具之间的逻辑关系
3. 识别哪些工具是SQE日常工作直接相关的

**深入学习**：详细学习请参考 [[06-AIAG五大工具]]

**五大工具关系图**：

```
APQP（产品质量先期策划）
    ↓
FMEA（失效模式与影响分析）
    ↓
CP（控制计划）
    ↓
MSA（测量系统分析）+ SPC（统计过程控制）
    ↓
PPAP（生产件批准程序）
```

**SQE相关度**：

| 工具 | SQE日常相关度 | 说明 |
|------|---------------|------|
| APQP | ★★★☆☆ | 了解供应商产品开发流程 |
| FMEA | ★★★★☆ | 审核必查，能识别风险点 |
| CP | ★★★★★ | 核心工具，必须精通 |
| MSA | ★★★★☆ | 审核必查，测量系统可靠性 |
| SPC | ★★★★☆ | 审核必查，过程能力判断 |
| PPAP | ★★★★★ | 核心工具，来料批准标准 |

**验证方式**：能解释FMEA和SPC在供应商审核中的应用场景

#### W4：零跑内部质量体系文件学习

**学习任务**：
1. 获取零跑质量体系文件清单
2. 优先学习：来料检验规程、供应商分级标准、质量问题处理流程
3. 识别内部术语与国标术语的对应关系

**零跑质量文件优先级**：

| 优先级 | 文件类型 | 说明 |
|--------|----------|------|
| P0 | 来料检验规程（IQC） | SQE日常核心工作 |
| P0 | 供应商分级标准 | 决定审核频率和方式 |
| P1 | 质量问题处理流程（8D） | 问题升级和处理路径 |
| P1 | 供应商审核规范 | VDA 6.3如何落地 |
| P2 | PPAP批准流程 | 新供应商/新产品导入 |

**验证方式**：能说明零跑来料检验的关键检查点

## 第2个月：供应商审核方法

### 里程碑目标

- 能完成一份供应商审核检查表
- 能独立进行过程审核（观摩后辅助主导）
- 能识别供应商现场的关键风险点

### 学习内容

#### W5-W6：VDA 6.3审核方法论

**深入学习**：详细VDA 6.3审核方法请参考 [[07-VDA6.3审核方法]]

**VDA 6.3核心内容**：

| 章节 | 内容 | SQE重点 |
|------|------|---------|
| P1 | 过程审核策划 | 审核计划如何制定 |
| P2 | 过程审核实施 | 审核检查表如何设计 |
| P3 | 过程审核评价 | 打分逻辑和判定标准 |
| P4 | 过程审核员能力 | 审核员资质要求 |
| 提问目录 | 7大过程领域 | 100+审核提问 |

**7大过程领域（提问目录）**：

```
P2.1 过程输入（供应商能力评估）
P2.2 策划（APQP执行情况）
P2.3 过程开发（试产验证）
P2.4 过程实现（批量生产控制）
P2.5 过程输出（出货质量）
P2.6 过程反馈（客户投诉处理）
P2.7 过程改进（纠正措施验证）
```

**审核检查表模板**：

| 序号 | 审核提问 | 证据来源 | 判定 | 备注 |
|------|----------|----------|------|------|
| 1 | 是否有质量管理体系证书？ | 证书原件 | OK/NOK | 有效期核查 |
| 2 | FMEA是否按要求更新？ | 最新FMEA文件 | OK/NOK | 版本日期核查 |
| 3 | 关键设备是否有点检记录？ | 点检表 | OK/NOK | 最近3个月 |
| ... | ... | ... | ... | ... |

**验证方式**：基于零跑供应商，完成一份审核检查表

#### W7：审核实操观摩

**观摩任务**：
1. 参与1-2次实际供应商审核（观摩角色）
2. 记录审核员的提问方式和记录方法
3. 识别现场观察要点（5S、工装设备、在制品管理）

**现场观察要点清单**：

| 观察点 | 关注内容 | 异常信号 |
|--------|----------|----------|
| 5S | 现场清洁度、物料定位 | 杂乱、地面油污 |
| 工装设备 | 点检记录、设备状态 | 点检缺失、设备故障 |
| 在制品 | 流转状态、半成品标识 | 无标识、混放 |
| 返工品 | 处理流程、区域隔离 | 随意放置、无记录 |
| 人员 | 操作是否符合作业指导书 | 操作与SOP不符 |
| 文件 | 版本有效性、记录完整性 | 版本过期、空白记录 |

**验证方式**：提交观摩记录，包含至少5个现场发现点

#### W8：辅助主导审核

**任务**：
1. 在资深SQE带领下，辅助完成部分审核内容
2. 独立完成检查表中的2-3个模块
3. 撰写审核发现记录

**审核发现分类**：

| 分类 | 说明 | 示例 |
|------|------|------|
| MAJOR | 重大不符合 | 缺失FMEA、关键特性无控制 |
| MINOR | 轻微不符合 | 点检记录不及时、文件版本未更新 |
| OBS | 改进建议 | 5S可提升、建议增加防错措施 |
| OK | 符合 | 无异常发现 |

**验证方式**：提交审核报告（至少包含5个发现点）

## 第3个月：问题处理基础

### 里程碑目标

- 能独立发起和跟踪质量问题
- 能用8D方法处理简单质量问题
- 能识别质量问题中的关键数据

### 学习内容

#### W9-W10：8D问题解决法

**8D步骤**：

```
D1：成立团队
D2：描述问题（what/where/when/extent）
D3：实施临时约束（containment）
D4：根本原因分析（5Why、鱼骨图）
D5：选择永久纠正措施
D6：实施永久纠正措施
D7：预防再发措施
D8：团队认可（Lessons Learned）
```

**D1-D3：问题描述与临时约束**

**学习任务**：
1. 理解3W2H1E问题描述法
2. 学习临时约束措施的选择逻辑
3. 练习：基于真实来料不良，完整描述问题

**问题描述模板**：

```
问题：XXX不良率超过标准值
何地：XXX供应商/XXX生产线
何时：2026-XX-XX首次发现
何事：不良表现为XXX，影响XXX
程度：当前不良率X%，标准值X%
```

**D4-D8：根因分析与纠正**

**学习任务**：
1. 掌握5Why根本原因分析
2. 学习鱼骨图（人机料法环测）
3. 区分纠正措施与预防措施

**5Why练习案例**：

```
问题：方向盘传感器来料不良率高
Why1：为什么不良率高？因为焊接不良
Why2：为什么焊接不良？因为焊锡膏活性不足
Why3：为什么活性不足？因为库存超期
Why4：为什么库存超期？因为没有先进先出管理
Why5：为什么没有先进先出？因为仓库没有FIFO制度
根因：仓库缺少FIFO管理制度

正确纠正措施：建立仓库FIFO管理制度
错误纠正措施：更换焊锡膏供应商（只解决一时）
```

**验证方式**：独立完成一个8D报告，通过直属领导评审

#### W11：质量问题处理案例分析

**任务**：
1. 研究零跑历史质量问题处理案例
2. 识别案例中8D应用的正误
3. 提炼SQE处理质量问题的标准话术

**标准话术模板**：

| 场景 | 话术 |
|------|------|
| 要求供应商回复 | "请在X个工作日内提交D3临时约束措施，X月X日前完成D4根因分析" |
| 升级问题 | "当前问题已超时X天未解决，申请升级处理" |
| 推动改进 | "基于本次问题，建议供应商在X方面进行改进，我会在下次审核中验证" |
| 解释8D价值 | "8D不是为了让你们写报告，而是帮助你们真正找到根因，避免重复发生" |

#### W12：主导第一个质量问题

**任务**：
1. 选择一个真实质量问题主导处理
2. 完成从问题描述到D8的全流程
3. 提交复盘报告

**复盘报告模板**：

```
问题概述：[一句话描述]
处理周期：X天
关键数据：不良率从X%降到X%
根因：XXX
纠正措施：XXX
预防措施：XXX
经验教训：下次遇到类似问题，应该XXX
```

### 常见误区（5个）

| 误区 | 正确理解 |
|------|----------|
| 认为审核就是找茬 | 审核是帮供应商发现问题，是合作关系 |
| 认为8D就是写报告 | 8D是思维方法，报告只是载体 |
| 以为标准越高越好 | 质量要与成本平衡，过度质量是浪费 |
| 认为来料检验合格=质量合格 | 过程能力比终检更重要 |
| 以为供应商问题都是供应商责任 | SQE有责任推动改进，OEM也需反思要求合理性 |
