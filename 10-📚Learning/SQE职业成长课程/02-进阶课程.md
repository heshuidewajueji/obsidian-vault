# SQE进阶课程：供应商质量管理深化

## 课程定位

**目标人群**：完成基础课程，能够独立负责日常审核和质量问题处理

**前置要求**：基础课程全部通过，具备3-6个月实战经验

**学习目标**：建立跨部门协调能力 + 系统性风险管理能力 + 推动供应商持续改进

---

## 学习路径

```
第7-9个月：系统风险管理与跨部门协调
第10-12个月：供应商战略合作与持续改进
```

---

## 第7-9个月：系统风险管理与跨部门协调

### 里程碑目标

- 能建立供应商质量风险评估模型
- 能协调多方资源推动供应商改进
- 能主导跨部门质量会议

### 学习内容

#### W25-W28：供应商质量风险评估

**风险评估模型**：

| 评估维度 | 权重 | 评估要点 |
|----------|------|----------|
| 历史质量表现 | 30% | 不良率、8D响应速度、PPAP通过率 |
| 审核结果 | 25% | VDA 6.3评分、改进措施落实情况 |
| 技术能力 | 20% | FMEA深度、设备能力、过程能力指数Cpk |
| 商务风险 | 15% | 单一供应商、产能风险、财务状况 |
| 合作意愿 | 10% | 响应速度、沟通配合度 |

**供应商分级决策矩阵**：

```
                  质量表现好
                       │
    ┌──────────────────┼──────────────────┐
    │                  │                  │
 战略级            优化级              问题级
 供应商            供应商              供应商
    │                  │                  │
    │           质量表现一般         质量表现差
    │                  │                  │
    └──────────────────┼──────────────────┘
                       │
                  需改进
```

**风险触发机制**：

| 触发条件 | 风险等级 | 应对措施 |
|----------|----------|----------|
| 连续2批次不良率>1000ppm | 🔴高 | 立即启动8D，暂停供货 |
| 审核评分<60分 | 🟡中 | 重点跟踪月度审核 |
| Cpk<1.33关键特性 | 🔴高 | 要求供应商提交改进计划 |
| 8D响应超时>5工作日 | 🟡中 | 发送正式通知 |
| 单一供应商物料 | 🔴高 | 推动备选供应商开发 |

**验证方式**：为负责的供应商建立完整风险档案（至少3家）

#### W29-W32：跨部门协调能力

**SQE需要协调的部门**：

| 部门 | 协调场景 | 关键成功因素 |
|------|----------|--------------|
| 采购 | 供应商选择、绩效评价、商务谈判 | 用数据说话，建立互信 |
| PE/工艺 | 技术问题分析、产品变更 | 技术文档规范化 |
| 生产 | 来料异常处理、换线验证 | 快速响应机制 |
| 质量/QE | 标准定义、重大问题升级 | 清晰的升级路径 |
| 物流 | 紧急物料调拨、质量隔离 | 库存数据透明 |

**跨部门会议话术模板**：

| 会议类型 | 目标 | 话术框架 |
|----------|------|----------|
| 周例会 | 同步状态，识别风险 | "本周重点：3个问题在跟踪，2个需要升级，1个已关闭" |
| 问题升级会 | 推动资源 | "问题影响X个车型，不良率X%，已超时X天，需要X部门支持" |
| 供应商绩效评审 | 季度评价 | "本季度评分X分，较上季度X%（上升/下降），主要差异是..." |

**验证方式**：主导1次跨部门质量会议，形成会议纪要并跟踪闭环

## 第10-12个月：供应商战略合作与持续改进

### 里程碑目标

- 能建立供应商长期合作关系
- 能设计并推动供应商改进项目
- 能建立个人SQE知识管理体系

### 学习内容

#### W33-W36：供应商战略合作

**供应商合作关系层级**：

```
Level 1：交易关系（价格导向）
Level 2：合作关系（质量导向）
Level 3：战略伙伴（共同成长）
Level 4：深度协同（系统集成）
```

**推动供应商升级策略**：

| 当前层级 | 目标层级 | 策略 | 时间周期 |
|----------|----------|------|----------|
| Level 1 | Level 2 | 建立质量KPI，月度review | 6-12个月 |
| Level 2 | Level 3 | 联合改进项目，技术输出 | 12-24个月 |
| Level 3 | Level 4 | 信息系统对接，JIT供货 | 24个月以上 |

**供应商赋能项目**：

| 赋能类型 | 内容 | 效果衡量 |
|----------|------|----------|
| 技术培训 | FMEA/SPC专项培训 | 供应商审核评分提升 |
| 过程审核 | 联合审核，输出改进建议 | 供应商Cpk提升 |
| 质量管理共建 | 帮助供应商建立IATF体系 | 减少重复问题发生 |
| 新产品导入 | APQP流程共建 | PPAP一次性通过率提升 |

**验证方式**：推动1家供应商从Level 1升级到Level 2

#### W37-W40：持续改进项目

**改进项目选择标准**：

| 维度 | 权重 | 评估要点 |
|------|------|----------|
| 改善潜力 | 30% | 当前不良率/成本，改进空间多大 |
| 可行性 | 25% | 供应商能力，资源投入 |
| 优先级 | 25% | 对零跑车型的影响程度 |
| 可见性 | 20% | 改善效果是否可量化 |

**改进项目实施框架**：

```
Step 1：现状测量（基线数据）
Step 2：目标设定（SMART）
Step 3：原因分析（5Why+FMEA）
Step 4：改善方案设计
Step 5：实施与监控（SPC）
Step 6：效果验证（前后对比）
Step 7：标准化与推广
```

**改善效果量化模板**：

```
改进前（基线）：XXX不良率X ppm，月度质量成本X万元
改进后：XXX不良率X ppm，月度质量成本X万元
改善幅度：不良率下降X%（X→X ppm），质量成本降低X万元/年
```

**验证方式**：完成1个供应商改进项目，有量化改善数据

#### W41-W48：个人SQE知识管理体系

**知识管理目标**：建立可传承、可复用的SQE知识资产

**知识库结构**：

| 模块 | 内容 | 更新频率 |
|------|------|----------|
| 供应商档案 | 基础信息、审核记录、问题历史 | 实时更新 |
| 问题案例库 | 典型问题分类整理，解决方案索引 | 每月新增 |
| 审核知识库 | 检查表模板、审核发现案例库 | 每季度更新 |
| 标准知识库 | IATF/VDA/ISO条款解读 | 按需更新 |
| 个人复盘 | 工作总结、经验教训 | 每周更新 |

**知识复利效应**：
- 第1年：建立档案，学习积累
- 第2年：复用档案，提升效率
- 第3年：形成方法论，输出赋能

**验证方式**：建立个人SQE知识库，包含：
- 5家以上供应商完整档案
- 10个以上问题案例
- 1套可复用的审核模板

| 误区 | 正确理解 |

## 主动输出任务（L1）

> **设计理念**：以教促学，通过输出验证深度理解。每项任务需产出具体交付物。

### 任务清单

#### 任务1：供应商质量风险评估报告
- **目标**：为3家供应商建立完整风险档案
- **交付物**：风险评估报告（含评分表、风险等级、改进建议）
- **模板**：
  ```
  供应商名称：____________
  评估维度得分：
  - 历史质量表现（30%）：___分
  - 审核结果（25%）：___分
  - 技术能力（20%）：___分
  - 商务风险（15%）：___分
  - 合作意愿（10%）：___分
  综合得分：___分 | 风险等级：🔴🟡🟢
  主要风险点：____________
  应对措施：____________
  ```

#### 任务2：跨部门协调会议主导
- **目标**：主导1次跨部门质量会议
- **交付物**：会议议程 + 会议纪要 + 跟踪清单
- **要求**：
  - 会前：提前1天发送议题
  - 会中：控制时间，达成共识
  - 会后：24小时内发送纪要，明确行动项和责任人

#### 任务3：供应商升级方案设计
- **目标**：为1家Level 1供应商设计升级路径
- **交付物**：升级方案PPT（现状分析→目标设定→实施路径→资源需求→时间表）
- **评估标准**：方案可行性、逻辑完整性

#### 任务4：供应商改进项目报告
- **目标**：完成1个供应商改进项目
- **交付物**：改进项目报告（含基线数据、改善效果、标准化建议）
- **模板**：
  ```
  问题描述：____________
  基线数据（改进前）：___不良率，___ppm，___万元/月
  目标设定：____________
  根本原因分析（5Why）：____________
  改善措施：____________
  改善效果（改进后）：___不良率，___ppm，___万元/月
  改善幅度：不良率下降___%，质量成本降低___万元/年
  标准化建议：____________
  ```

#### 任务5：个人SQE知识库搭建
- **目标**：建立个人SQE知识管理体系
- **交付物**：
  - 5家供应商完整档案
  - 10个问题案例（含问题描述、根因、解决方案）
  - 1套可复用审核模板
- **工具建议**：Obsidian/Notion/Excel均可

### 输出检验标准

| 任务 | 优秀（90分） | 合格（60分） | 待改进（<60分） |
|------|-------------|-------------|----------------|
| 风险评估报告 | 评分完整、有数据支撑、措施具体 | 评分基本完整 | 评分缺失或措施不可行 |
| 会议主导 | 议程清晰、纪要规范、跟踪闭环 | 能完成基本流程 | 会议无结论或无跟踪 |
| 升级方案 | 可操作、有时间表、有资源规划 | 方案基本可行 | 目标不清晰或路径不可行 |
| 改进项目 | 有量化数据、逻辑清晰、可复制 | 有改善数据 | 无数据或无改善 |
| 知识库 | 结构完整、内容充实、便于检索 | 基本框架有内容 | 内容残缺或无结构 |
