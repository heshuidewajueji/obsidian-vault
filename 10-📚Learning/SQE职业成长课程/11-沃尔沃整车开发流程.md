# 11-沃尔沃整车开发流程

> 来源：沃尔沃汽车产品开发流程及整车项目计划
> 适合：项目经理、SQE、质量工程师

## 沃尔沃开发流程核心理念

### 安全性第一（Safety First）
- 安全是沃尔沃品牌核心
- 所有开发决策以安全为前提

### 质量门体系（Gate Review）
- 每个阶段设置质量门
- Gate Review通过才能进入下一阶段
- 强调：DVP（设计验证计划）+ PVS（生产验证阶段）

---

## 整车开发阶段划分

### Phase 0: 规划阶段
- 市场调研
- 概念设计
- 可行性分析
- 预算评估

### Phase 1: 概念阶段
- 目标定义
- 方案选择
- 虚拟验证
- 成本目标设定

### Phase 2: 开发阶段
```
子系统开发 → 系统集成 → 整车集成
    ↓           ↓           ↓
 零部件验证   系统验证   整车验证
```

### Phase 3: 验证阶段
- 整车测试（DVP&R）
- 安全碰撞测试
- 寒热耐候测试
- 量产准备

### Phase 4: 投产启动
- 试生产
- 爬坡量产
- SOP（批量生产启动）

---

## 沃尔沃质量门体系

| Gate | 内容 | 交付物 |
|-----|------|-------|
| G1 | 规划门 | 可行性报告、项目章程 |
| G2 | 概念门 | 概念设计、目标成本 |
| G3 | 开发门 | 设计验证完成 |
| G4 | 产品门 | DVP完成、SOP准备 |
| G5 | 量产门 | 产能爬坡完成 |

| 里程碑 | 说明 |
|-------|------|
| PRR | 项目启动评审 |
| SFR | 设计冻结 |
| PVS | 预生产 |
| OS | 零批量 |

| MP | 大批量生产 |

## 沃尔沃项目计划结构

### 时间维度
```
方案阶段（6-8月）→ 设计阶段（12-18月）→ 验证阶段（12-18月）→ 投产阶段（6-12月）
```

### 关键路径
1. 造型冻结 → 工程设计 → 样件制作 → 测试 → 设计变更
2. 供应商定点 → 模具开发 → 零部件验证 → 量产准备

## 跨部门协同机制

### 核心委员会
- **SE board**：系统工程委员会
- **VA/VE委员会**：价值分析/价值工程
- **质量委员会**：跨部门质量问题解决

### 会议机制
| 会议 | 频率 | 参与者 |
|-----|------|-------|
| 项目例会 | 每周 | PM + 核心团队 |
| 技术评审 | 每阶段 | 相关部门 |
| 质量门评审 | 按Gate | 高层 + 质量 |

## 对SQE的启示

### 供应商介入时机
- 概念阶段：供应商技术交流
- 设计阶段：SQE早期介入
- 验证阶段：问题跟踪、8D推动
- 量产阶段：日常审核、持续改进

### 关键控制点
1. 供应商PPAP提交审核
2. 来料检验与SPC监控
3. 客户投诉处理闭环
4. 供应商年度审核

## Anki记忆卡（L2）

### Card 1：GVDP阶段与Gate对应关系
> Q：Phase 2开发门(G3)通过的交付物是什么？
> A：DVP（设计验证计划）完成 + 子系统验证报告

### Card 2：关键里程碑顺序
> Q：汽车开发中PRR→SFR→PVS→SOP的正确顺序是什么？
> A：PRR(项目启动)→SFR(设计冻结)→PVS(预生产)→SOP(批量生产启动)

### Card 3：SQE在Phase 2的核心任务
> Q：概念阶段到开发阶段，SQE早期介入的主要工作？
> A：供应商技术交流、DVP审核、推动8D解决质量问题

### Card 4：质量门数量与命名
> Q：GVDP中从规划到量产共设置几个质量门？
> A：5个质量门：G1规划门→G2概念门→G3开发门→G4产品门→G5量产门

### Card 5：PVS与SOP的区别
> Q：PVS（预生产验证阶段）和SOP（批量生产启动）的核心差异？
> A：PVS验证产品设计成熟度，SOP验证生产过程成熟度

### Card 6：跨部门委员会职责
> Q：SE board、VA/VE委员会、质量委员会各自职责？
> A：SE board=系统工程决策；VA/VE=成本与价值优化；质量委员会=跨部门质量问题解决
