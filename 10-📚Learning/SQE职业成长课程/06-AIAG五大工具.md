## 一、五大工具总览

### 工具定位

| 工具 | 全称 | 核心作用 | 所属阶段 |
|------|------|----------|----------|
| APQP | 产品质量先期策划 | 结构化开发流程，串联其他四大工具 | 贯穿整个APQP |
| FMEA | 失效模式与影响分析 | 风险预防，识别潜在失效模式 | 设计+过程开发 |
| MSA | 测量系统分析 | 确保测量数据准确可靠 | 量产前+量产中 |
| SPC | 统计过程控制 | 过程能力监控，变差控制 | 量产阶段 |
| PPAP | 生产件批准程序 | 量产准入批准，验证过程能力 | 量产前批准 |

### 五大工具逻辑关系图

```
APQP（产品质量先期策划）——框架性方法论
    ↓
    ├──→ FMEA（缺陷预防工具）→ 过程设计和开发阶段
    │
    └──→ 控制计划CP（核心工具）→ 串联以下三个工具
                    │
                    ├──→ MSA（测量系统分析）→ 确保检测有效
                    ├──→ SPC（统计过程控制）→ 确认指标满足
                    └──→ PPAP（量产前批准）→ 最终批准
```

## 二、APQP 产品质量先期策划

### 2.1 概述

**定义**：用来确定和制定确保某产品使顾客满意所需步骤的结构化方法
**版本**：AIAG APQP第2版（2008年7月）
**核心目标**：促进与所涉及的每个人的联系，确保要求的步骤按时完成
**关注点**：T（时间）、Q（质量）、C（成本）的平衡

### 2.2 五阶段模型

```
阶段1：计划和确定项目
    ↓ 输入：顾客呼声、业务计划、产品/过程基准数据
阶段2：产品设计和开发
    ↓ 输入：设计目标、可靠性和质量目标、初始材料清单
阶段3：过程设计和开发
    ↓ 输入：DFMEA、控制计划、新设备/工装要求
阶段4：产品和过程确认
    ↓ 输入：试生产控制计划、MSA计划、PPK研究
阶段5：反馈、评定和纠正措施
    ↓ 输出：持续改进、顾客满意
```

### 阶段1：计划和确定项目

**输入**：
- 顾客呼声（市场研究、保修记录、质量信息）
- 业务计划/营销策略
- 产品/过程基准数据
- 产品/过程设想
- 产品可靠性研究
- 顾客输入

**输出**：
- 立项表/试作通知书
- 产品质量先期策划进度表
- 多功能小组任命
- 顾客图纸/样品

### 阶段2：产品设计和开发

**核心输出**：
- DFMEA（设计FMEA）
- 可制造性和装配设计
- 设计验证/设计评审
- 工程图样、规范、材料规范
- 样件控制计划

### 阶段3：过程设计和开发

**核心输出**：
- PFMEA（过程FMEA）
- 过程流程图
- 特性矩阵图
- 试生产控制计划
- 过程指导书
- MSA计划和PPK研究计划

### 阶段4：产品和过程确认

**核心输出**：
- 有效生产运行
- 测量系统分析（MSA）
- 初始过程能力研究（PPK/Cpk）
- 生产确认试验
- 包装评价
- 生产控制计划
- PPAP提交

### 阶段5：反馈、评定和纠正措施

**核心输出**：
- 削减变差（SPC数据应用）
- 顾客满足
- 交付和服务
- 持续改进

### 2.3 APQP与SQE工作关系

**SQE关注重点**：
- 供应商是否按APQP阶段执行
- 关键节点是否留有记录
- 特殊特性是否在控制计划中标注
- FMEA是否动态更新

## 三、FMEA 失效模式与影响分析

### 3.1 概述

**定义**：潜在失效模式及影响分析，用于识别产品和过程中的潜在失效模式
**类型**：
- DFMEA：设计FMEA（Design），针对产品设计阶段
- PFMEA：过程FMEA（Process），针对生产过程阶段
**版本**：AIAG FMEA第4版（2008年6月）

### 3.2 FMEA基本结构

**分析项目**：
1. 潜在失效模式（Failure Mode）：以什么方式失效
2. 潜在失效后果（Effect）：失效对顾客的影响
3. 严重度S（Severity）：后果严重程度（1-10分）
4. 潜在失效原因（Cause）：失效发生的起因
5. 频度O（Occurrence）：原因发生的频率（1-10分）
6. 现行设计控制（Current Design Controls）：现有检测手段
7. 探测度D（Detection）：失效被探测的难度（1-10分）
8. 风险优先数RPN（Risk Priority Number）：S×O×D

### 3.3 评分标准

**严重度S评分（10分制）**：

| 评分 | 影响描述 | 示例 |
|------|----------|------|
| 10 | 灾难性（安全/法规违规） | 转向失效导致事故 |
| 9 | 很高（产品功能丧失） | 安全气囊失效 |
| 7 | 高（系统性能下降） | ABS功能异常 |
| 5 | 中等（部分功能受影响） | 仪表显示异常 |
| 3 | 轻微（可识别但影响小） | 外观瑕疵 |
| 1 | 无影响 | 无任何影响 |

**频度O评分（10分制）**：

| 评分 | 频度等级 | 失效比率（近似） |
|------|----------|-----------------|
| 10 | 极高 | ≥1/2 |
| 7 | 高 | 1/20 |
| 5 | 中等 | 1/500 |
| 3 | 低 | 1/2000 |
| 1 | 极低 | ≤1/1500000 |

**探测度D评分（10分制）**：

| 评分 | 探测难度 | 示例 |
|------|----------|------|
| 10 | 几乎不可能 | 无检测手段 |
| 7 | 很难探测 | 特定测试条件才能发现 |
| 5 | 中等难度 | 常规检测可发现 |
| 3 | 较易探测 | 目视检查可发现 |
| 1 | 容易探测 | 自动检测100%覆盖 |

**RPN = S × O × D**
- 取值范围：1～1000
- RPN阈值（传统标准）：≥100需立即采取纠正措施
- 现代方法更推荐使用AP（动作优先级）矩阵

### 3.4 FMEA与SQE工作关系

**审核关注点**：
- 供应商是否有FMEA台账
- FMEA是否为动态文件（随项目进展更新）
- 高RPN项是否有对应纠正措施
- 纠正措施是否已实施并验证
- 特殊特性的S≥8项是否设置了监控计划

## 四、MSA 测量系统分析

### 4.1 概述

**定义**：对每个零件能够重复读数的测量系统进行分析，评定测量系统的质量
**版本**：AIAG MSA第4版（2010年6月）
**核心指标**：GRR（ Gage R&R，重复性和再现性）

### 4.2 测量系统五特性

| 特性 | 定义 | 评价方法 |
|------|------|----------|
| 偏性（Bias） | 测量平均值与基准值的差异 | 与标准件对比 |
| 重复性（Repeatability） | 同一人、同一仪器、同一零件多次测量差异 | EV（设备变差） |
| 再现性（Reproducibility） | 不同人、同一仪器测量平均值差异 | AV（评价者变差） |
| 线性（Linearity） | 偏性随测量范围的变化 | 全量程对比 |
| 稳定性（Stability） | 测量系统随时间的保持能力 | 多次校准对比 |

### 4.3 GRR分析

**GRR = EV + AV**（重复性 + 再现性合成）

**评价标准**（%GRR基于过程变异）：

| %GRR范围 | 判定 | 行动 |
|----------|------|------|
| <10% | 可接受 | 无需改进 |
| 10%~30% | 临界 | 依据特性重要度和成本决策 |
| >30% | 不可接受 | 必须改进 |

**简化评价标准**（AIAG传统标准）：

| %GRR范围 | 判定 |
|----------|------|
| <10% | 可接受 |
| 10%~20% | 基本可接受 |
| 20%~30% | 可能可接受，需依据成本和重要性决策 |
| >30% | 不可接受，必须改进 |

**GRR实验设计**：
- 评价者：2-3人（典型为3人）
- 零件数：10个（覆盖全范围）
- 测量次数：每人2-3次
- 总数据量：2人×10零件×2次 = 40个数据

### 4.4 MSA与SQE工作关系

**审核关注点**：
- 供应商是否对关键特性量具进行了GRR分析
- GRR结果是否满足要求（%GRR<10%）
- 测量系统分辨率是否为过程变差的1/10
- 超出控制限的量具是否经过修调后方可使用

## 五、SPC 统计过程控制

### 5.1 概述

**定义**：应用统计分析技术对生产过程进行适时监控，科学区分随机波动与异常波动
**版本**：AIAG SPC第2版（2005年6月）
**核心控制图**：Xbar-R图（均值-极差图）、p图、u图等

### 5.2 过程能力指数

**基本概念**：
- USL：规格上限（Upper Specification Limit）
- LSL：规格下限（Lower Specification Limit）
- σ（标准差）：过程固有变差

**Cp（过程潜力指数）**：
```
Cp = (USL - LSL) / 6σ
```
衡量过程的潜在能力，假设过程均值在规格中心

**Cpk（过程能力指数）**：
```
Cpu = (USL - X̄) / 3σ
Cpl = (X̄ - LSL) / 3σ
Cpk = min(Cpu, Cpl)
```
考虑过程均值偏移的实际能力

**Pp/Ppk**：用于初始过程研究（试产阶段），对应长期能力

### 5.3 Cpk评价标准

| Cpk值 | 过程能力等级 | 判定 |
|-------|------------|------|
| ≥1.67 | 优秀 | 无需调整 |
| ≥1.33 | 充足 | 可接受 |
| ≥1.0 | 勉强 | 需要监控改进 |
| <1.0 | 不足 | 不可接受 |

**汽车行业典型要求**：Cpk ≥ 1.33（关键特性）or Cpk ≥ 1.0（一般特性）

### 5.4 控制图判异准则

**8条判异准则**（Western Electric Rules）：
1. 1点落在控制限之外
2. 连续9点落在中心线同一侧
3. 连续6点递增或递减
4. 连续14点交替上下
5. 连续3点中有2点落在2σ区域外
6. 连续5点中有4点落在1σ区域外
7. 连续15点落在1σ区域内（过度聚集）
8. 连续8点落在1σ区域外

### 5.5 SPC与SQE工作关系

**审核关注点**：
- 供应商是否对关键特性实施SPC控制
- 控制图是否实时更新
- 异常点是否有原因分析和纠正记录
- Cpk是否满足顾客要求
- 超控制限产品是否经过隔离和评估

## 六、PPAP 生产件批准程序

### 6.1 概述

**定义**：用来确定供应商是否正确理解了顾客工程设计记录和规范的要求，以及生产过程是否具有潜在能力
**版本**：AIAG PPAP第4版（2006年3月）
**提交时机**：新产品首次供货、工程变更、二级供应商变更等

### 6.2 提交等级

| 等级 | 要求 | 说明 |
|------|------|------|
| 等级1 | 仅提交保证书+外观批准报告（如适用） | 最简化 |
| 等级2 | 保证书+产品样品+有限支持资料 | 次简化 |
| 等级3 | 保证书+产品样品+完整支持资料 | **默认等级**（常用） |
| 等级4 | 保证书+顾客规定的其他要求 | 定制化 |
| 等级5 | 保留在供应商制造厂备查（保证书+样品+完整资料） | 审核模式 |

**零跑/大多数OEM默认使用等级3**

### 6.3 必须提交文件清单（等级3）

| 序号 | 文件名称 | 提交要求 |
|------|----------|----------|
| 1 | 零件提交保证书（PSW） | S（必须提交） |
| 2 | 设计记录（工程图样） | S |
| 3 | 工程变更文件（如有） | S |
| 4 | 工程批准（如要求） | S |
| 5 | DFMEA（如有） | S |
| 6 | 过程流程图 | S |
| 7 | PFMEA | S |
| 8 | 控制计划 | S |
| 9 | 测量系统分析研究（MSA） | S |
| 10 | 全尺寸测量结果 | S |
| 11 | 材料/性能试验结果 | S |
| 12 | 初始过程研究（Cpk/Ppk） | S |
| 13 | 合格实验室文件 | S |
| 14 | 外观批准报告（如适用） | S |
| 15 | 生产件样品 | S |
| 16 | 标准样品 | R（保留） |
| 17 | 检查辅具 | R |
| 18 | 顾客特殊要求记录 | R |

**S = 必须提交；R = 保留在现场，顾客要求时提供**

### 6.4 初始过程研究要求

**Cpk/Ppk判定标准**：
- Cpk ≥ 1.67：该生产节拍的过程能力优秀
- Cpk ≥ 1.33：该过程能力满足要求
- Cpk ≥ 1.0：勉强接受，但需改进
- Cpk < 1.0：不接受，需供应商提交纠正措施

**数据要求**：
- 数据来源：连续生产（至少300件，或1小时生产量）
- 25组数据，每组至少4个样品
- 过程必须稳定（无异常波动）

### 6.5 PPAP与SQE工作关系

**审核关注点**：
- 供应商是否按时提交完整PPAP文件包
- Cpk是否满足要求
- DFMEA/PFMEA是否与实际一致
- MSA是否覆盖关键特性
- PSW是否由供应商授权人签字

## 七、五大工具实战关联

### 7.1 供应商审核中的应用场景

| 场景 | 关注工具 | SQE行动 |
|------|----------|----------|
| 新供应商准入 | APQP+FMEA+PPAP | 审核APQP执行情况，检查PPAP提交 |
| 量产过程监控 | SPC+MSA | 验证SPC控制图有效性，审核MSA GRR |
| 质量问题处理 | FMEA+SPC | 用FMEA识别风险，用SPC数据验证改善 |
| 工程变更 | PPAP | 变更后重新提交PPAP |

### 7.2 SQE审核检查表要点

**APQP审核**：
- [ ] 是否有APQP小组和进度表
- [ ] 各阶段输入输出是否完整
- [ ] 特殊特性是否在FMEA和CP中标识
- [ ] FMEA是否为动态文件

**FMEA审核**：
- [ ] S≥8的项是否有应急响应措施
- [ ] 高RPN项是否采取了纠正措施
- [ ] 纠正措施是否已验证有效

**MSA审核**：
- [ ] 关键特性量具是否有GRR报告
- [ ] %GRR是否<10%
- [ ] 量具分辨率是否满足要求

**SPC审核**：
- [ ] 关键特性是否有控制图
- [ ] 是否有判异规则培训记录
- [ ] Cpk是否达到1.33

**PPAP审核**：
- [ ] 文件包是否完整（18项）
- [ ] Cpk/Ppk是否≥1.33
- [ ] PSW是否签字完整

## 八、常见误区

| 误区 | 正确理解 |
|------|----------|
| FMEA只是设计阶段的工作 | FMEA应动态更新，量产发现的问题应反馈到FMEA |
| SPC是质检员的事 | SPC是过程控制工具，操作工和技术员都应参与 |
| MSA只做一次 | 新设备、新量具、量产初期都需进行MSA |
| PPAP只是填表 | PPAP是验证供应商过程能力的工具，核心是Cpk数据 |
| 五大工具独立使用 | 五大工具是一个系统，APQP是框架，串联其他工具 |

### FMEA计算题

**题目**：某零部件的失效模式分析得到以下数据：
- 潜在失效后果严重度 S = 8
- 潜在失效原因频度 O = 5
- 现行探测措施探测度 D = 4

请计算：
1. RPN = ?
2. 如果将探测措施改进使D降至2，RPN变为多少？改善幅度？
3. 根据AIAG标准，该RPN水平需要采取纠正措施吗？

---

### MSA分析题

**题目**：某供应商对关键尺寸量具进行了GRR研究，结果如下：
- 设备变差 EV = 10%
- 评价者变差 AV = 8%

请回答：
1. %GRR = ?
2. 该测量系统是否可接受？
3. 如果AV过高，作为SQE应建议供应商如何改进？

---

### SPC过程能力题

**题目**：某零件规格限为 50±0.5mm，测量25组数据后计算得到：
- 过程均值 X̄ = 50.02mm
- 过程标准差 σ = 0.08mm

请计算：
1. Cp = ?
2. Cpu、Cpl、Cpk = ?
3. 该过程能力是否满足汽车行业≥1.33的要求？

---

### PPAP提交清单题

**题目**：某供应商准备提交PPAP，你作为SQE审核其文件包。

3. 外观批准报告（非外观件）

## Anki记忆卡（L2）

> 碎片化记忆，巩固核心概念

### 卡片组1：五大工具定位

```
正面：APQP的核心作用是什么？
背面：产品质量先期策划的框架性方法论，串联其他四大工具，贯穿从概念设计到量产的全过程。

正面：FMEA中RPN如何计算？
背面：RPN = S（严重度）× O（频度）× D（探测度），取值范围1-1000，≥100需立即纠正。

正面：MSA中%GRR的可接受标准是多少？
背面：%GRR < 10% 可接受；10%~30% 临界；>30% 不可接受需改进。
```

### 卡片组2：过程能力指数

```
正面：Cp和Cpk的区别是什么？
背面：Cp = (USL-LSL)/6σ，假设过程均值居中，衡量潜在能力；
      Cpk = min(Cpu,Cpl)，考虑均值偏移，实际能力。
      Cpk ≤ Cp，当均值偏离中心时差距越大。

正面：汽车行业关键特性的Cpk要求是多少？
背面：Cpk ≥ 1.33（关键特性），Cpk ≥ 1.0（一般特性）。

正面：Ppk和Cpk有什么区别？
背面：Ppk用于初始过程研究（试产阶段），长期能力；
      Cpk用于量产过程，已知过程均值和标准差。
```
