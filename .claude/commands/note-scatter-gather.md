# 碎片笔记整合命令

## 任务
当用户有一个模糊的主题方向时，扫描 Vault 收集所有相关碎片，整合成完整的知识网络。

## 执行步骤

### Phase 1: 主题探索
1. 理解用户的核心问题或主题
2. 生成相关关键词列表
3. 扩展关联概念

### Phase 2: 碎片收集
- 搜索所有包含相关关键词的笔记
- 提取每个碎片中的核心观点
- 记录碎片的来源和上下文

### Phase 3: 模式识别
- 找出碎片之间的共同主题
- 识别矛盾的观点
- 发现隐含的结论

### Phase 4: 整合输出
- 按逻辑顺序组织碎片
- 生成 MOC 草稿
- 识别可以合并的笔记

## 输出格式

```markdown
# [主题] 碎片整合报告

## 收集到的碎片（按来源）
### 来源1: [[笔记A]]
- 观点：...
- 关键词：...

### 来源2: [[笔记B]]
- 观点：...
- 关键词：...

## 发现的主题模式
1. 模式A（X 个碎片支持）
2. 模式B（X 个碎片支持）

## 矛盾观点
- 观点1 vs 观点2：...

## 隐含结论
[从碎片中浮现的结论]

## 建议的 MOC 结构
- 主笔记：[[主题]]
  - 子主题1：[[相关笔记链接]]
  - 子主题2：[[相关笔记链接]]
```

## 使用场景
```
/note-scatter-gather [主题关键词]
```
