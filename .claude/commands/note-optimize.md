# 笔记优化命令

## 任务
分析当前 Vault 的结构健康度，识别需要优化的笔记和链接问题。

## 执行步骤

### Phase 1: 结构扫描
1. 统计笔记总数和文件夹分布
2. 识别孤立笔记（无入链/无出链）
3. 检查断链和悬空引用
4. 分析标签使用情况

### Phase 2: 问题识别
- 找出超过 30 天未更新的笔记
- 识别内容重复或高度相似的笔记
- 发现潜在的 MOC 需求

### Phase 3: 优化建议
- 给出具体的整理建议
- 标记可以合并的笔记
- 建议需要补充链接的位置

## 输出格式

```markdown
# Vault 健康报告

## 基本统计
- 总笔记数：
- 文件夹数：
- 平均链接数：

## 问题笔记
### 孤立笔记（需关注）
| 笔记名 | 状态 | 建议 |

### 长期未更新（可能已过时）
| 笔记名 | 最后修改 | 天数 |

### 断链列表
| 位置 | 断链目标 |

## 优化建议
1. [优先级高]
2. [优先级中]
3. [优先级低]
```

## 使用场景
```
/note-optimize
```
